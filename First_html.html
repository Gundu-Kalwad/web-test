Q1. What is HTML? Explain features of HTML.

Answer — definition, features and explanation.

Definition

HTML stands for HyperText Markup Language. It is the standard markup language used to create and structure content displayed in web browsers. HTML uses tags (elements) to tell the browser how to present texts, images, links and other content.

Detailed Explanation

HTML is not a programming language; it is a markup language. An HTML file contains elements such as headings, paragraphs, lists, tables, forms and media. Browsers parse the HTML and render the page visually. HTML works together with CSS (for styling) and JavaScript (for interactivity).

Features of HTML
  • Simple and Easy: Easy to learn and write even for beginners.
  • Platform Independent: Runs on any OS and browser.
  • Hyperlinking: Supports linking between pages using the <a> tag.
  • Multimedia Support: Can embed images, audio and video (<img>, <audio>, <video>).
  • Structured Documents: Headings, paragraphs, lists and tables allow clear structure.
  • Extensible & Evolving: HTML5 added semantic elements, canvas, local storage and more.
  • Integrates with CSS/JS: Separate presentation and behavior make development flexible.
Conclusion

HTML forms the backbone of the web. Its features make it suitable for building both simple static pages and complex modern web applications when combined with CSS and JavaScript.

Q2. What is an HTML Document?

Answer — structure and characteristics of an HTML document.

Definition

An HTML document is a text file with a .html or .htm extension that contains HTML markup. Browsers read and render it as a webpage.

Key Characteristics
  • Starts with the declaration <!DOCTYPE html> (HTML5).
  • Has a root element: <html></html>.
  • Two main parts: <head> (metadata) and <body> (visible content).
  • Contains a <title> shown in browser tab and other metadata such as <meta> tags.
Example: Basic HTML Document
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>My First Webpage</title>
  </head>
  <body>
    <h1>Welcome to HTML</h1>
    <p>This is a simple HTML document.</p>
  </body>
</html>
Conclusion

An HTML document is the file the browser uses to build and display a webpage. Proper structure ensures correct rendering across browsers.

Q3. Explain the basic structure of HTML document.

Answer — step-by-step explanation of the skeleton of an HTML page.

Basic Structure Elements
  1. <!DOCTYPE html> — Declares HTML5 and helps browsers render correctly.
  2. <html> — The root element (contains head and body).
  3. <head> — Contains metadata: <title>, <meta>, styles and links.
  4. <body> — Contains all visible content (headings, paragraphs, images, etc.).
Example & Explanation
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>Example Page</title>
    <meta name="description" content="Short description">
  </head>
  <body>
    <h1>Main Heading</h1>
    <p>Paragraph content here.</p>
  </body>
</html>
Why this structure matters

Correct structure helps search engines, accessibility tools, and browsers understand and render your content. The <head> holds resources and metadata, while the <body> is what users see.

Conclusion

Adhering to the basic structure ensures compatibility, correct rendering and better SEO/accessibility.

Q4. Explain the following tags with an example: Header tag, Paragraph tag, Subscript & Superscript

Answer — definitions, use-cases, examples and notes on semantics & accessibility.

A. Header Tags (<h1> to <h6>)

Header tags define headings and subheadings. Use them in hierarchical order (one <h1> per page for main title, then <h2>, <h3>, ...). Search engines and screen readers use heading structure to understand page content.

B. Paragraph Tag (<p>)

The paragraph element is used for blocks of text. Browsers apply default margins which separate content and improve readability.

C. Subscript (<sub>) and Superscript (<sup>)

Use <sub> to show text slightly below baseline (commonly used in chemical formulas). Use <sup> to show text above baseline (exponents, ordinal indicators).

Rendered examples:

Chemical formula: H2O — the 2 appears below the line.

Mathematics: X2 + Y2 — exponents appear above the line.

Best Practices
  • Follow heading hierarchy for accessibility.
  • Use semantic tags (<strong> / <em>) instead of purely presentational tags when possible.
  • Use sub/sup only for semantic requirements (not to fudge spacing or font sizes).
Conclusion

Header tags give structure and SEO value; paragraphs organize text; sub/sup are required for scientific and mathematical correctness and accessibility.

Q5. Explain an HTML element where we demonstrate working with text tags.

Answer — full description of text formatting tags with examples and semantic differences.

Definition & Purpose

HTML text formatting elements change the appearance or emphasize text. They fall into two categories:

  • Presentational (physical): <b>, <i>, <u> — focus on how text looks.
  • Semantic (logical): <strong>, <em>, <mark> — convey meaning; better for accessibility and SEO.
Detailed Examples
Rendered demonstration:

Bold using <b> (visual)

Strong using <strong> (semantic + visual)

Italic using <i> (visual)

Emphasized using <em> (semantic)

Underlined using <u>

Highlighted using <mark>

Small text using <small>

<p><strong>Important:</strong> Please read the instructions carefully.</p>
<p><em>Note:</em> This field is required.</p>
<p><mark>Reminder:</mark> Submit before the deadline.</p>
When to use semantic vs presentational tags

Prefer semantic tags when the text meaning matters (screen readers, search engines). Use presentational tags only when you explicitly need a visual effect and there's no semantic meaning.

Conclusion

Text formatting improves readability and conveys meaning; semantic tags support accessibility and search engines while presentational tags control appearance.

Q6. Explain <table> tag along with its specific properties.

Answer — structure, attributes, examples showing rowspan/colspan, and best practices.

Definition

The <table> element displays tabular data with rows and columns. It is composed of rows (<tr>), header cells (<th>) and data cells (<td>).

Common Attributes
  • border — legacy attribute to set border width (use CSS border instead in modern pages).
  • cellpadding — internal cell padding (use CSS padding nowadays).
  • cellspacing — gap between cells (use CSS border-spacing).
  • rowspan & colspan — merge cells vertically or horizontally.
Example (with rowspan and colspan)
Rendered table example:
Name Age Course
Rahul 20 B.Sc
Priya 21
End of Table 
<table border="1" cellpadding="5" cellspacing="3">
  <tr>
    <th>Name</th>
    <th>Age</th>
    <th>Course</th>
  </tr>
  <tr>
    <td>Rahul</td>
    <td>20</td>
    <td rowspan="2">B.Sc</td>
  </tr>
  <tr>
    <td>Priya</td>
    <td>21</td>
  </tr>
  <tr>
    <td colspan="3">End of Table</td>
  </tr>
</table>
Best Practices
  • Use CSS for styling (borders, spacing, widths) instead of deprecated attributes.
  • Use <caption> for table titles and <thead>, <tbody>, <tfoot> for semantics.
  • Provide scope attributes on <th> for accessibility when appropriate.
Conclusion

Tables are ideal for tabular data. Use semantic markup and CSS for styling and ensure the table is accessible to screen reader users.

Q7. What is a list? Explain different types of lists.

Answer — ordered, unordered and definition lists with examples and use-cases.

Definition

A list groups related items. HTML provides three types of lists to show ordered, unordered or term-definition data.

Types of Lists

1. Ordered List (<ol>)

Displays items with numbers or letters. Use when sequence matters (e.g., steps).

<ol>
  <li>Step 1: Open the browser</li>
  <li>Step 2: Type the URL</li>
  <li>Step 3: Press Enter</li>
</ol>

2. Unordered List (<ul>)

Displays items with bullets. Use when order is not important (e.g., shopping lists).

<ul>
  <li>Milk</li>
  <li>Bread</li>
  <li>Butter</li>
</ul>

3. Definition List (<dl>)

Used for pairs of terms and descriptions (glossary entries).

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
</dl>
Conclusion

Use ordered lists for sequences, unordered lists for grouped items, and definition lists for term–definition relationships. Lists improve readability and content organization.

Q8. Explain different form controls and elements with an example.

Answer — common form controls, how they behave, and a complete sample form.

Definition

HTML forms collect user input. The form is the container: <form> ... </form>. Inside it you place input controls such as text fields, radio buttons, checkboxes, selects and buttons.

Common Form Controls and their Roles
  • <input type="text"> — single-line text box.
  • <input type="password"> — hides typed characters for passwords.
  • <input type="radio"> — choose one option in a group.
  • <input type="checkbox"> — choose multiple options.
  • <select><option> — dropdown selection.
  • <textarea> — multi-line text input.
  • <button type="submit"> / <input type="submit"> — submit the form.
